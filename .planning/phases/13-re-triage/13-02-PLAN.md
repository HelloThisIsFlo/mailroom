    1. Replace `_check_already_grouped()` with `_detect_retriage(self, sender: str) -> tuple[str | None, str | None]`:
       - Search for contact via `self._carddav.search_by_email(sender)`
       - If no results: return (None, None)
       - Extract contact_uid from vCard
       - Call `self._carddav.check_membership(contact_uid)` WITHOUT exclude_group parameter
       - Return (contact_uid, group_name) if in a group, or (contact_uid, None) if not in any group

    2. Add `_reconcile_email_labels(self, sender: str, category: ResolvedCategory, add_to_inbox: bool) -> int`:
       - Compute managed_mailbox_names: `{c.destination_mailbox for c in self._settings.resolved_categories}`
       - Convert to IDs: `{self._mailbox_ids[name] for name in managed_mailbox_names}`
       - Get screener_id: `self._mailbox_ids[self._settings.triage.screener_mailbox]`
       - Fetch all sender emails: `self._jmap.query_emails_by_sender(sender)`
       - If no emails, return 0
       - Get per-email mailbox membership: `self._jmap.get_email_mailbox_ids(all_email_ids)`
       - Compute new destination IDs using get_parent_chain (same logic as _get_destination_mailbox_ids but takes category directly):
         ```python
         resolved_map = {c.name: c for c in self._settings.resolved_categories}
         chain = get_parent_chain(category.name, resolved_map)
         new_dest_ids = [self._mailbox_ids[c.destination_mailbox] for c in chain]
         ```
       - Build per-email patch dict:
         - For each email: remove all managed_mailbox_ids + screener_id (set to None), add new_dest_ids (set to True)
         - For add_to_inbox: if `add_to_inbox` is True AND email's current mailboxIds contains screener_id, also add Inbox ID
         - Inbox is NEVER set to None (never removed)
       - Execute patches in BATCH_SIZE chunks via self._jmap.call() with Email/set, same error handling pattern as batch_move_emails
       - Return count of emails reconciled

    3. Add `_reassign_contact_groups(self, contact_uid: str, old_category: ResolvedCategory, new_category: ResolvedCategory) -> None`:
       - Compute chain diff:
         ```python
         resolved_map = {c.name: c for c in self._settings.resolved_categories}
         old_chain = get_parent_chain(old_category.name, resolved_map)
         new_chain = get_parent_chain(new_category.name, resolved_map)
         old_groups = {c.contact_group for c in old_chain}
         new_groups = {c.contact_group for c in new_chain}
         ```
       - Add to new-only groups FIRST: `for g in new_groups - old_groups: self._carddav.add_to_group(g, contact_uid)`
       - Then remove from old-only groups: `for g in old_groups - new_groups: self._carddav.remove_from_group(g, contact_uid)`

       ```

         - contact_uid set, old_group set -> is_retriage=True
       Step 3: Upsert contact with triage history note (same call, now produces history format)
       Step 4: Contact group management
         - If is_retriage: find old_category from old_group, call _reassign_contact_groups(contact_uid, old_category, category)
         - If initial triage: add to ancestor groups (existing behavior, unchanged)
       Step 5: Email label management
         - If is_retriage: call _reconcile_email_labels(sender, category, category.add_to_inbox)
         - If initial triage: sweep Screener only (existing behavior, unchanged)

         - If is_retriage: log "group_reassigned" with old_group, new_group=group_name, same_group=(old_group==group_name), emails_reconciled
         - If initial triage: log existing "triage_complete" event (unchanged)
       Step 7: Remove triage label -- LAST STEP (unchanged)
       ```
